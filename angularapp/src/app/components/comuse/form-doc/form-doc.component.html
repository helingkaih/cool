表单组件使用说明

什么是表单组件

点击查看官方说明

用表单处理用户输入是许多常见应用的基础功能。 应用通过表单来让用户登录、修改个人档案、输入敏感信息以及执行各种数据输入任务。

系统的表单三要素：FormData、FormGroup、FormOrder

系统中是怎么使用表单组件的

  • 首先从后端获取到表单数据 (大部分是后端传来,少数是前端组的表单数据) 。
  • 传来的表单数据不能直接使用 (点击查看数据说明) ,需要使用指定的 Service 处理一遍。
  • 处理完后的表单数据可以使用，为了验证和提交数据方便，我们还会将处理完的数据使用指定方法组成 FormGroup

为什么会有个新的 FormService

点击查看新 FormService 说明

  • ( 以下新 FormService 简称 新方法，旧的简称 旧方法 )。
  • 旧方法存在时间久，修改过的人多，不断会有新的需求导致要修改这个方法，以至于现在旧方法代码很冗余，处理很繁杂，性能也就差了。
  • 由于旧方法的性能差、可读性差、可维护性差等几方面的因素，诞生了个新方法。
  • 新方法代码精简，性能高，只处理必须要的数据、可拓展性高，目前正在慢慢适配整个系统。
  • 为了保证新方法不会发展成旧方法那样，如果需要修改到新方法时，必须找相关负责人协商。

什么是 FormGroup

点击查看官方说明

FormGroup 把每个子 FormControl 的值聚合进一个对象， 它的 key 是每个控件的名字。 它通过归集其子控件的状态值来计算出自己的状态。 比如，如果组中的任何一个控件是无效的，那么整个组就是无效的，我们提交表单数据的时候会使用其 value 属性。 总的来说 FormGroup 就是表单数据的总控中心，可以实现非常多的操作。

什么是 FormOrder

FormOrder 顾名思义，就是表单的顺序，用来在页面上按顺序展示表单字段
