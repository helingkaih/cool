sc-web 规范说明

目标：清晰、简洁、易懂、可拓展、可复用、高性能

HTML 规范

调整好多层嵌套的 HTML 标签的格式，根据情况再选择换行

合适的注释，合适的间距

禁止单独一个标签写 for/if，尽量使用 ng-container 或者直接在目标元素上写 for/if

TS 规范

没有特殊情况的话，所有组件都要开启 OnPush 模式

除了生命周期，其他所有方法都必须写注释，且方法长度尽量控制在75行以内

合理的封装复用代码

这一点是毋庸置疑的，在同一个功能模块下很大概率会出现相同的逻辑处理现象，这个时候对代码进行合理的封装会提高开发效率和代码的易读性，但需要控制，不能过度封装。

Css 规范

部分主要样式需要用系统封装好的, 点击查看系统主要样式说明

为了避免重复定义样式，造成代码冗余混乱，系统规定了部分经常使用的样式，并已经全局注册，直接使用即可。

少用或不用 important

important 属性过于霸道，过多使用的话一旦出问题，那将会是场噩梦，提高了样式的维护难度。

要覆盖子组件的样式时，需要用子代选择器，并且父样式是当前父组件的

可能会有这么一种情况，一个页面使用到了两个相同的子组件，其中一个样式要变，那就得在父组件相应位置增加一个父样式，这样一来就可以单独控制该父样式下子组件的样式了
